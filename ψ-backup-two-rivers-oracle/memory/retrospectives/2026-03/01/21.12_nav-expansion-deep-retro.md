### 3. Scroll Spy Rewrite (IO → Scroll Position)
```javascript
// Before: IntersectionObserver (broken for nested sections)
const spy = new IntersectionObserver((entries) => {
  entries.forEach(e => { if (e.isIntersecting) setActive(...); });
}, { threshold: 0.1 });

// After: Scroll-position math (handles nesting correctly)
function updateSpy() {
  var active = 0;
  for (var i = sections.length - 1; i >= 0; i--) {
    if (sections[i] && sections[i].getBoundingClientRect().top <= 120) {
      active = i; break;
    }
  }
  setActive(active);
}
window.addEventListener('scroll', function() {
  if (!rafId) rafId = requestAnimationFrame(updateSpy);
}, { passive: true });
```

## Architecture Decisions

### IntersectionObserver vs Scroll Position — Decision Tree

**Use IntersectionObserver when**: detecting single element entry/exit, scroll reveals, lazy loading, "load more" triggers, contextual button visibility (Report button pattern).

**Use scroll-position math when**: determining which of N sections is "active" (scroll spy), nested DOM hierarchy where parent/child overlap in viewport, needing exactly one winner.

**Root cause of IO failure**: The `#landings`, `#student-oracles`, `#senior-projects` divs are children of `#workshop`. When any subsection is visible, workshop is also visible. IO fires for the most recently changed entry — but when scrolling up between adjacent subsections, neither fires (both already intersecting). The active state gets stuck.

The scroll-position approach iterates bottom-to-top and picks the last section whose top has passed 120px from viewport top. It handles all scroll directions and nesting correctly.

### Why 120px Threshold

The fixed navbar sits at `top: 16px` with `padding: 12px 24px`, making it roughly 56px tall. With 16px top offset, the navbar extends to ~72px. The 120px threshold adds ~48px breathing room so the section has clearly entered the viewport before becoming active. This matches the `scroll-margin-top: 80px` on subsections.

## Lessons Learned

### 1. IntersectionObserver is Wrong for Nested Section Scroll Spy
**Confidence**: High (verified in production)

IO fires based on threshold crossings. When child sections are nested inside a parent, the parent remains "intersecting" indefinitely while children overlap. Scrolling UP between adjacent children produces no state change — both are already intersecting. The active dot freezes.

**Rule**: Use `getBoundingClientRect().top <= threshold` with reverse iteration for scroll spy when sections are nested or can overlap in the viewport.

### 2. Sub-Dot Visual Hierarchy is a 3-Line Pattern
**Confidence**: High (deployed, looks right)

```css
.dot-nav .dot.sub { width: 7px; height: 7px; border-width: 1.5px; }
```

Communicates parent/child relationship without any text, tooltips, or indentation. Apply `class="sub"` to subsection dots. Active state applies identically to both sizes.

### 3. scroll-margin-top is the Correct Anchor Offset
**Confidence**: High (CSS spec, no ambiguity)

Don't use padding-top hacks or negative margin tricks for fixed-navbar anchor offset. `scroll-margin-top: 80px` is purpose-built for this. Works with `scroll-behavior: smooth`. No layout side effects.

### 4. The Deploy-to-Test Cycle is a Solved Problem
**Confidence**: High (4 retros have identified it)

`python3 -m http.server 8080` in `docs/` gives identical results to GitHub Pages for static HTML/CSS/JS. No build step differs. Zero setup required. Saves 30-60 seconds per iteration and prevents noisy commit history.
